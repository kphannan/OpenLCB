NMRA 2009 Demos

Brief description of proposed demos for NMRA 2009 summer meeting with a more detailed To-Do list and notes here

Constraints:

  • Portable

  • Limited amount of hardware available

  • Want to demonstrate real working system and hopefully knock their socks off...

  • Here's a link to Bob's Demo Hardware that he will take to the Convention

Possible Demos

1) Basic hardware


Show that initialization of multiple nodes doesn't require anything. Add the 3rd node “out of box”, show that it starts up and can be easily configured.



Illustration 1: Two shields demonstrating Blue/Gold programming.



2) Computer Interface




Add a computer via USB connection and show that everything still works. Demonstrate computer responding to hardware events, and hardware responding to computer events. Key to do this demonstration second, as the first example doesn't need a computer in any way.

3) Ethernet Connectivity




Demo Ethernet connection by adding a computer via ethernet-CAN adapter; show that's also plug-and-play.

Diagram shows two computers, is that needed?

4) Ethernet Bridging for Larger Layouts




Two CAN segment communicating over Ethernet without a PC doing any intervention, e.g. for a modular layout setup.

Here is a picture of Bob's portable demo system of Demo 4




5) LocoNet Connectivity




Not sure of topology of this one, basic idea is to have LocoNet events coexist with P/C NMRAnet events, and to move LocoNet from one place to another over an NMRAnet backbone. Limited by available hardware?
(dph: I would suggest a Loconet-Ethernet Gateway for this, if possible. )

Needed Hardware

What hardware is needed?

Three LEDuinos fitted out to demo I/O (at least two switches/pushbuttons, two LEDs, a few more is better; some need buttons/LEDs for blue/gold too)

Two LEDuinos with Ethernet hardware

Ethernet switch, cables, etc

One LEDunio for USB-CAN adapter (could be same as one of the above?)

Two LEDuinos for LocoNet interfaces (could be some of above; not sure whether it's impressive to load new code for new function on fly during demo or not)

Two passive LocoNets: cables, power

Two SE8c boards with LEDs (masts) and switches/buttons

Power supplies

Two sets CAN cabling
