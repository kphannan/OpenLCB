OpenLCB Node ID Specification

Introduction (Informative)

This specification describes the format and allocation of OpenLCB node identifers (node IDs). It is not specific to any wire protocol.

References and Context (Normative)

Format

An OpenLCB node identifier (node ID) shall be six bytes of eight bits each.

The order of bytes in an OpenLCB node ID shall be considered significant. The most-significant byte shall be transmitted first during communication operations. The most-significant byte shall be written first (left-most in Western format) in any human-readable representation.

OpenLCB node IDs shall include one or more 1 bits.
