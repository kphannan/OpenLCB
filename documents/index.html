Notes

Background

Non-normative notes containing background information on decisions, etc.

  • Numerology - discussion of reasoning behind decisions on field sizes

  • Message Format - discussion of reasoning behind message format decisions

  • Optional Interactions, uses for streaming (should be moved to other pages)

  • Routing - development of routing protocol

  • Message Types for Simple Nodes - how simple nodes can interact with a subset of the protocol

  • External Connections - how to map and connect NMRAnet to external systems like DCC, XPressNet, Loconet, etc

  • Use With DCC - discussion of adapters between NMRAnet and DCC controls

  • Signaling - discussion of using NMRAnet for layout train signals

  • Discovery of Topology - discussion of how to discover the module connections in a large modular layout, and what to do with that information

  • Locomotive Control – looking into how to eventually implement locomotive control (throttles) over NMRAnet

  • Configuration – development of protocol for configuring nodes

  • Role of Ethernet – role of direct (not via IP) Ethernet connectivity

  • On Software Structure – Some discussion of how various structures for computer programs work with NMRAnet

  • On Using Producer/Consumer Events– Discussion of how Producer/Consumer Events can be used on a model railroad.

  • On Layout ID – discussion of the need for a unique “Layout ID” string

  • Logging – a proposal for a common error logging and retrieval technique

  • Crossing Gate example – how a model railroader can first encounter NMRAnet when building a crossing gate
