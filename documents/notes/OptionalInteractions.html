Optional Interactions

These interactions need not be implemented. If they are, they must work at least as described here.

Configuration

Determining the type information from a node is a standard (required) interaction. Doing the configuration via the OpenLCB is optional, but most vendors will want to provide it.

Basic architecture: configuration is a vector of setting values, which is described by higher-level format information that's available directly from the node. The configuration vector doesn't have to be organized the same as it's stored in the device, but it certainly can be.

Do we want to provide both a low-rate, simple protocol plus a higher-speed streaming version?

Read Configuration Format

Uses a stream to pull data describing the content of the configuration date.

Read Configuration Data

Uses a stream to pull all the configuration data.

Write Configuration Data

Uses a stream to put a new set of configuration data.

Streaming Interactions

OpenLCB provides a streaming protocol for moving large quantities of data. Nodes must implement the standard interaction for streaming, which is just enough to gracefully turn down a streaming request. They may optionally implement streaming for one or more of several purposes.
