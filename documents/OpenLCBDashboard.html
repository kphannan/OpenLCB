Event Transport


Fixed length identifiers that can propagate from multiple source nodes to multiple destination nodes. These are typically used to announce that something of interest has happened on the layouts so that controlled devices can respond.


Datagram Transport


Move up to 84 bytes efficiently from a source node to a specific
target node.



CAN Datagram Transport

Protocol adapted to CAN, including fragmentation and defragmentation of datagrams into CAN-frames.


Stream Transport


An efficient way to move a large, variable number of bytes from one node to another.






Application Layer

PIP Protocol Identification Protocol

This protocol allows a node to determine another node's capabilities, which is reflected by the protocols it understands.


Producer Consumer Protocol

Work-horse protocol covering the sending of events, and the classification and documenting of events handled by nodes.


Memory Configuration Protocol

Reading and writing from node memory in terms of overlapping memory spaces.
